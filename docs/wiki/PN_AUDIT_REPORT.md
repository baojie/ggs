# PN 引注全量核查与批量修复报告

## 背景

KLG4 评估显示约 30% 的 PN 引注存在偏差（偏移或凭空引用）。需要建立系统化流程来
核查并修复全部 1332 条引注。

## 流程

```
pn_audit.py                          pn_repair_pipeline.py
    │                                     │
    │  Stage 1: 全量扫描                  │
    │  ├── 遍历 247 页                   │
    │  ├── 提取 1332 条 PN 引注          │
    │  └── 自动分类 OK/偏移/原文无        │
    │                                     │
    │  Stage 2: 偏移二审确认 (--stage2)   │
    │  ├── 仅保留 ±1 范围的偏移           │
    │  ├── 核验建议PN的bigram匹配更优      │
    │  └── 289 确认 + 21 驳回             │
    │                                     │
    │  Stage 3: 批量修复 (--stage3)       │
    │  ├── 按slug分组，同页多处一次处理    │
    │  ├── 上下文感知定位（处理同名多现）  │
    │  ├── 直接替换 + record_revision     │
    │  └── 285/289 修复成功 (98.6%)       │
    │                                     │
    │  Stage 4: 待审报告 (--report)       │
    │  └── 输出 pn_human_review.md        │
    │                                     │
    │  Stage 5: 复检 (--stage5)           │
    │     └── 验证新旧PN是否如预期替换     │
    │                                     │
    └────────────────── 修复管线已执行 ────→
```

## 现有 PN 基因分析

### 现有基因覆盖（13+ 个）

| 基因 | 功能 | 粒度 |
|------|------|------|
| QUO6 pn_verify | 单页面核验，高置信度自动修正 | 单页 |
| QUO10 cross-chapter | 跨章语义一致性 | 单条引注 |
| QUO23 inline-pn-assertion-verify | 行内PN断言核验 | 单条引注 |
| QUO24 pn-accuracy-audit | 全库blockquote批量扫描，仅报告不修复 | 全库 |
| FIX9 quote-attribution-audit | 检测并修复/删除幻觉引文 | 单条引注 |
| CHK8 citation-validity | 五维合法性门控 | 提交关卡 |
| COR9 offline-citation-integrity | 字符串级存在性验证 | 单条引注 |

### 缺口

1. **缺乏端到端批量修复管线**：QUO24 做全量扫描但不修复；FIX9 做修复但不批量。
2. **上下文感知修复缺失**：现有基因遇到同页多处同名PN时无法准确定位。
3. **无修复后自动验证**：QUO6/FIX9 修复后不验证结果。
4. **待审报告输出缺失**：无法批量输出的待人工审核清单。

### 改进方案

建议在现有基因基础上新增或增强：

**新增 QUO25 — Bulk PN Audit Pipeline**

```
QUO25-bulk-pn-audit.md
├── 上游依赖: QUO7（格式校验）、FIX10（格式修复）、PRE20（PN source构建）
├── 核心步骤:
│   ├── 1. 全量扫描（复用 QUO24 的扫描逻辑）
│   ├── 2. 自动分类（OK / 偏移 / 原文无）
│   ├── 3. 偏移二审确认（仅 ±1）
│   ├── 4. 上下文感知批量修复
│   ├── 5. 修复后复检
│   └── 6. 输出待审报告
├── 调用脚本: pn_audit.py + pn_repair_pipeline.py
└── 输出: pn_audit_report.json + pn_human_review.md
```

**增强现有基因**：

- `QUO24`: 增加 `--fix` 模式，调用 QUO25 的修复管线
- `QUO23`: 增加 assertion context-aware 定位能力
- `FIX9`: 增加批量模式（batch-mode），支持一次性处理全库偏移
